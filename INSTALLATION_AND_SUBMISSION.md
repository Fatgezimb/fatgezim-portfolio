# Installation and Submission Options

## Option A — Attach the pack to a Codex project

This is the safest option when the repository already has its own `AGENTS.md`.

1. Keep this folder intact.
2. Attach the folder or ZIP to the Codex project.
3. Attach selected files from `visuals/`.
4. Paste `SUBMIT_THIS_TO_CODEX.md`.
5. Ask Codex to read this pack as a specification and merge—not overwrite—existing repository guidance.

## Option B — Copy selected files into the repository

Copy:

- `prompts/`
- `project-briefs/`
- `design/`
- `schemas/`
- `checklists/`
- `.agents/skills/portfolio-phase-gate/`
- `PLANS.md`
- `phase-state.json`

For `AGENTS.md`:

- If the repository does not have one, copy this pack's file.
- If it already has one, merge the mandatory phase gate and source-of-truth rules into the existing file.
- Do not erase valid build commands or repo-specific instructions.

## Option C — Use only the master prompt

Paste `prompts/00_MASTER_ORCHESTRATOR.md`, then paste `prompts/01_PHASE_1_CONTENT_LAYOUT.md`.

This works, but the skill, checklists, schemas, and persistent phase state make the workflow more reliable.

## Visual attachments

Attach, at minimum:

- `visuals/markups/01_desktop_content_layout.png`
- `visuals/markups/02_project_hover_storyboard.png`
- `visuals/markups/03_scroll_3d_motion_map.png`
- `visuals/markups/04_mobile_layout.png`
- one or two files from `visuals/renders/`

The generated concept boards contain placeholder copy and visualized metrics. They are style references only.
