# Project Selection Matrix

Score every candidate from 0–5 in each category. Record evidence for every score.

| Criterion | Weight | What a 5 means |
|---|---:|---|
| Evidence quality | 5 | Repo, screenshots, docs, links, and role are all clear |
| Relevance to current direction | 4 | Strongly supports neuroscience/medicine/ABA/data/software/founder narrative |
| Zim's contribution | 4 | Ownership and specific contribution are documented |
| Demonstrable workflow | 3 | Can show a truthful micro-demo without private data |
| Technical or domain depth | 3 | Shows meaningful complexity or expertise |
| Outcome clarity | 3 | Has a verifiable output, adoption state, or completed artifact |
| Differentiation | 2 | Memorable and unlike generic portfolio work |
| Visual potential | 2 | Can be communicated clearly and attractively |
| Public safety/privacy | 2 | Can be shown without PHI, secrets, or restricted data |
| Currentness | 2 | Represents current or strategically important work |

## Weighted score

For each row:

`score × weight`

Rank by total, then apply editorial judgment so the final set tells a coherent story.

## Selection rules

- Aim for four to six featured projects.
- Include at least one product/founder project.
- Include at least one deep technical, automation, data, or research project when evidence supports it.
- Include RBT/ABA work only with clear unofficial/educational boundaries.
- Do not include two projects that tell exactly the same story unless their audiences and contribution are meaningfully different.
- A named project may be excluded when evidence is weak.
- A previously unnamed project may be included when its evidence and relevance are stronger.
