# Featured Project Briefs and Micro-Demo Stories

These briefs summarize currently supported evidence. Codex must inspect the repository and source material for exact names, current URLs, technologies, dates, and status.

## 1. NeuroPathLabs / NeuroPath LLC

### Verified direction

Materials describe a NeuroPath organization and a training ecosystem connected to Bela Data Lab Academy. Current evidence includes team-led ABA training, caregiver learning, downloadable resources, completion/quiz states, certificate workflows, and public certificate verification.

### Naming caution

The source set contains both `NeuroPathLabs` and the legal name `NeuroPath LLC`. Determine the exact public brand before publishing.

### Strong portfolio angle

Founder-built behavioral-health learning and operations ecosystem that brings clinical knowledge, education, product design, and technology together.

### Candidate micro-demo

1. Open a training catalog.
2. Filter by learner type or topic.
3. Open a module.
4. Complete a short learning checkpoint.
5. Show progress/certificate readiness.
6. Display a “no PHI” product boundary.

Do not imply clinical treatment, certification authority, or BACB endorsement.

### Naming caution

## 3. RBT-Pro Prep / RBT Competency Prep

### Verified direction

An unofficial learning aid organized around all 19 RBT competency tasks, including assessment types such as With Client, Role-Play, and Interview. Existing planning includes a Competency Map, flashcards, practice questions, study guide, progress/review, and account/pricing concepts.

### Naming caution

Confirm whether the public name is `RBT-Pro Prep`, `RBT Competency Prep`, `NeuroPathLabs RBT Competency Prep`, or another verified variant.

### Strong portfolio angle

A structured competency-learning product that turns a complex professional checklist into a clear, trackable study journey.

### Candidate micro-demo

1. Show the 19-task competency map.
2. Select a task.
3. Flip or advance through one original flashcard.
4. Answer one original practice question.
5. Update a local progress state.
6. Show an explicit unofficial-learning-aid disclaimer.

Do not use BACB logos, copy proprietary test-bank text, or imply endorsement.

## 4. Rethink Automations

### Verified direction

A local Python web UI using Playwright over Chrome CDP. It manages Rethink workflow state, queued custom programs, behavior tasks, confirmation before automation, and a live/readable run log. Current evidence indicates Step 1 behavior automation with later steps represented as manual workflow state pending safe implementation.

### Strong portfolio angle

A practical automation system that converts a slow, fragile browser workflow into a reviewable, stateful operator dashboard with confirmation and diagnostics.

### Candidate micro-demo

1. Search/select a queued program or behavior.
2. Edit a safe local mock field.
3. Click Prepare Run.
4. Show page-confirmation state.
5. Simulate a staged local run.
6. Animate a readable log from pending → running → complete.
7. Clearly label the demo as sanitized and local.

Do not imply unsupported Steps 2–6 automation. Do not expose client data.

## 5. Bela Caregiver Library / Caregiver Academy

### Verified direction

A caregiver-focused learning library with practical ABA-informed modules, short video slots, scripts, flashcards, knowledge checks, routine maps, decision trees, trackers, and exportable tools. The tone is warm, practical, nonjudgmental, and explicitly educational rather than individualized care.

### Strong portfolio angle

A content-rich caregiver product that turns behavioral concepts into practical routines, tools, and plain-language support.

### Candidate micro-demo

1. Choose a routine or challenge.
2. Open a module.
3. Use one interactive tool, such as a routine map or communication menu.
4. Generate a small next-step plan.
5. Show a safety/when-to-seek-help boundary.
6. Show a static “not individualized care” note.

## 6. Research, neuroimaging, data, or medical projects

Codex must search the supplied résumé, repository, publications, posters, notebooks, data projects, and presentations.

A research/technical project should replace a product project when it scores higher on:

- evidence,
- relevance to the owner’s future direction,
- depth of contribution,
- visual demonstrability,
- credibility,
- differentiation.

### Candidate micro-demo pattern

- question/problem,
- shareable input,
- method,
- transformation/analysis,
- interpretable output,
- limitations.

Never expose restricted data or overstate findings.
